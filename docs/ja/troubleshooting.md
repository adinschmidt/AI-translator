# トラブルシューティング

## 設定画面で "Invalid API Endpoint URL" と表示される

エンドポイントが `https://`（ローカル Ollama の場合は `http://`）で始まる完全な URL であることを確認してください。推奨値に戻すには **Fill Default** を使用してください。

## 401 / 403 エラー

- 設定画面の API キーを再確認してください。
- プロバイダー側コンソールでキーが有効か確認してください。
- 一部プロバイダーでは、キーを利用する前に課金設定が必要です。

## レート制限またはクォータ超過エラー

プラン上限を超えると、プロバイダー側でリクエストが制限・拒否される場合があります。
時間を置いて再試行するか、プランのアップグレードを検討してください。

## Ollama モデルが読み込めない

- Ollama が起動しているか確認してください: `ollama serve`。
- 拡張機能からのリクエストを許可するため、`OLLAMA_ORIGINS="*"` を設定して再起動してください。
- 既定のエンドポイント `http://localhost:11434` を使用してください。

## ページ全体翻訳で表示が崩れる

ページ全体翻訳は実験的機能です。レイアウトが崩れた場合はタブを再読み込みして元の状態に戻し、まずは小さな選択範囲で翻訳を試してください。

## 拡張機能が反応しない

- `chrome://extensions/` または `about:debugging` で拡張機能を再読み込みしてください。
- **Options** ページを開き直し、設定が保存されているか確認してください。
