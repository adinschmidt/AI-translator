# 문제 해결

## 설정에서 "잘못된 API 끝점 URL"

엔드포인트가 `https://`(또는 로컬 Ollama의 경우 `http://`)으로 시작하는지 확인하세요.
전체 URL입니다. 권장 값을 복원하려면 **Fill Default**를 사용하세요.

## 401/403 오류

- 설정에서 API 키를 다시 확인하세요.
- 공급자 콘솔에서 키가 활성화되어 있는지 확인합니다.
- 일부 제공업체에서는 키를 사용하기 전에 결제를 활성화해야 합니다.

## 속도 제한 또는 할당량 오류

귀하가 계획 한도를 초과하는 경우 공급자는 요청을 제한하거나 차단할 수 있습니다.
나중에 다시 시도하거나 제공업체 요금제를 업그레이드하세요.

## Ollama 모델을 로드하지 못했습니다.

- Ollama이(가) 실행 중인지 확인하세요: `ollama serve`.
- 확장 요청을 허용하려면 `OLLAMA_ORIGINS="*"`을 설정하고 다시 시작하세요.
- 기본 엔드포인트 `http://localhost:11434`를 사용합니다.

## 전체 페이지 번역이 깨진 것 같습니다

전체 페이지 번역은 실험적입니다. 페이지 레이아웃이 깨지면 탭을 다시 로드하세요.
원본 콘텐츠를 복원하고 더 작은 선택 항목을 번역해 보세요.

## 확장 프로그램이 응답하지 않습니다.

- `chrome://extensions/` 또는 `about:debugging`에서 확장을 다시 로드합니다.
- **옵션** 페이지를 다시 열고 설정이 저장되었는지 확인하세요.
