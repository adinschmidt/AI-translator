# 故障排除

## 设置中的“API 端点 URL 无效”

确保端点以 `https://` 开头（或以 `http://` 表示本地 Ollama）
并且是一个完整的 URL。使用 **Fill Default** 恢复推荐值。

## 401 / 403 错误

- 仔细检查设置中的 API 密钥。
- 确认密钥在提供商控制台中处于活动状态。
- 一些提供商要求在使用密钥之前启用计费。

## 速率限制或配额错误

如果您超出计划限制，提供商可以限制或阻止请求。
稍后重试或升级您的提供商计划。

## Ollama 模型加载失败

- 确认 Ollama 正在运行：`ollama serve`。
- 设置 `OLLAMA_ORIGINS="*"` 并重新启动以允许扩展请求。
- 使用默认端点 `http://localhost:11434`。

## 整页翻译看起来很糟糕

整页翻译是实验性的。如果页面布局损坏，请重新加载选项卡
恢复原始内容并尝试翻译较小的选择。

## 扩展程序没有响应

- 在 `chrome://extensions/` 或 `about:debugging` 中重新加载扩展。
- 重新打开 **选项** 页面并确保保存设置。
